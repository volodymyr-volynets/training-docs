Introduction to Numbers Framework

Numbers Framework is a modular MVC (model-view-controller) framework built for PHP language.

It provides your eco-system with robust features, like:

  • Numbers ORM - an Object-Relational Mapper that makes it easy to interact with your database.
    1. Table Models - this is a main building block to define tables. Based on data defined in these models we generate migrations and can perform direct schema changes on databases.
    2. Active Records - each table model has one active record class associated with it. This class provides object oriented way of working with tables in database.
    3. Query Builder - a set of classes to generate and run SQL queries.
    4. Migrations - as part of the ORM you have a choice to use migrations or do direct schema changes for your database.
    5. Seeders - seed your database with fake data.
    6. SQL Objects - validate and generate complex SQL queries for different databases.
  • RBAC - a Role Based Access Control to easy manage your permissions.
    1. Users - manage direct user permissions known as user based access control (UBAC).
    2. Roles - list of roles and their permissions. Parent relations are available.
    3. Teams - list of teams and their permissions. Parent relations are available.
    4. Realms - list of realms and their permissions. Parent relations are available.
    5. Domains - list of domains and their permissions. Parent relations are available.
    6. Classifications - list of classifications and their permissions. Parent relations are available.
    7. Groups - create categories of users.
  • Form Builder - as part of a Framework and Frontend Repository use fully featured and easy to use builder for:
    1. Forms - build interactive forms.
    2. Lists - build interactive lists.
    3. Reports - build interactive reports.
    4. Collections - build multi form collections.
  • Localization and internalization - as part of a Internalization Repository use various localization, internalization and translation functionalities:
    1. Locales - a list of supported locales.
    2. Languages - a list of supported languages.
    3. Timezones - a list of supported timezones.
    4. Translations - uses text to text translations (i18n).
    5. Localizations - uses key to text translations (loc).
