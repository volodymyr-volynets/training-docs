Application Folders

Structure:

  1. Components - a list of classes to work with external libraries usually have frontend and PHP class to use that frontend.
  2. Config - a list of application configuration files stored in folders.
  3. Controller - a list of application controllers.
  4. Data - a list of import data files usually presets data in database for application to function properly.
  5. DataSource - a list of data sources that use query builder to query database.
  6. Form - a list of form, lists, reports and/or form collections.
  7. Helper - a list of helper classes for application to use.
  8. Layout - a list of application layout files usually HTML from templates to render on frontend and email templates.
  9. Miscellaneous - a list of classes grouped in folders usually data from Numbers Repositories is being prest to these folders:
    1. Commands - a list of available commands to be called from make command.
    2. Constants - a list of constants.
    3. Loads - lists of loaders.
    4. Localization - a list of localization files gathered from all Numbers Modules.
    5. Middlewares - a list of middlewares.
    6. Migrations - a list of migrations generated by all Numbers Modules.
    7. Routes - a list of available routes from all Numbers Modules.
    8. Seeders - a list of available seeders.
    9. UnitTests - unit tests
  10. Model - a list of models for application.
  11. Modules - This is folder to split application into modules.
  12. Overrides - application overrides.
  13. SQL - SQL files for the application.
  14. View - available views and templates.
