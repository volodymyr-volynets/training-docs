ORM Tables

Table classes reside in Model folder of a module or application. Changes to these classes generate changes to database objects in form of migration or direct schema changes.

For example if you use migrations and want to add a new column to the existing table you would need to modify columns property and then run:

        
            make migration_code_test verbose=1 # view changes
            make migration_code_commit # generate migration files
            make migration_db_commit # commit db changes

            namespace Numbers\Tenants\Tenants\Model;

            use Object\Table;

            class Tenants extends Table
            {
                public $db_link;
                public $db_link_flag;
                public $module_code = 'TM';
                public $title = 'T/M Tenants';
                public $name = 'tm_tenants';
                public $pk = ['tm_tenant_id'];
                public $tenant = false;
                public $orderby;
                public $limit;
                public $column_prefix = 'tm_tenant_';
                public $columns = [
                    'tm_tenant_id' => ['name' => 'Tenant #', 'domain' => 'tenant_id_sequence'],
                    'tm_tenant_code' => ['name' => 'Code', 'domain' => 'domain_part'],
                    'tm_tenant_name' => ['name' => 'Name', 'domain' => 'name'],
                    'tm_tenant_email' => ['name' => 'Primary Email', 'domain' => 'email', 'null' => true],
                    'tm_tenant_phone' => ['name' => 'Primary Phone', 'domain' => 'phone', 'null' => true],
                    'tm_tenant_tm_database_code' => ['name' => 'Database', 'domain' => 'database', 'null' => true],
                    'tm_tenant_um_regten_id' => ['name' => 'Registration #', 'domain' => 'group_id', 'null' => true],
                    'tm_tenant_inactive' => ['name' => 'Inactive', 'type' => 'boolean']
                ];
                public $constraints = [
                    'tm_tenants_pk' => ['type' => 'pk', 'columns' => ['tm_tenant_id']],
                    'tm_tenant_code_un' => ['type' => 'unique', 'columns' => ['tm_tenant_code']],
                    'tm_tenant_tm_database_code_fk' => [
                        'type' => 'fk',
                        'columns' => ['tm_tenant_tm_database_code'],
                        'foreign_model' => Databases::class,
                        'foreign_columns' => ['tm_database_code']
                    ],
                ];
                public $indexes = [
                    'tm_tenants_fulltext_idx' => ['type' => 'fulltext', 'columns' => ['tm_tenant_code', 'tm_tenant_name']]
                ];
                public $history = false;
                public $audit = false;
                public $optimistic_lock = true;
                public $options_map = [
                    'tm_tenant_name' => 'name',
                    'tm_tenant_code' => 'name',
                    'tm_tenant_id' => 'name'
                ];
                public $options_active = [
                    'tm_tenant_inactive' => 0
                ];
                public $engine = [
                    'MySQLi' => 'InnoDB'
                ];

                public $cache = true;
                public $cache_tags = [];
                public $cache_memory = false;

                public $data_asset = [
                    'classification' => 'proprietary',
                    'protection' => 1,
                    'scope' => 'global'
                ];
            }

Explanation of Tenants table model properties:

  1. $db_link and $db_link_flag - database links, determines which database connection to use. Set it for multi-database connection scenario. Default blank.
  2. $module_code - 2 letter module code, values are form sm_modules table.
  3. $title - model title.
  4. $name - database table name.
  5. $pk - list of primary key columns in order, used in get() and getStatic() methods and in Collections.
  6. $tenant - flag indicates that table fetches should be by tenant.
  7. $orderby - array of sorting columns, used for sorting in options() and Collections.
  8. $limit - you can limit of # of rows during fetches.
  9. $column_prefix - unique column prefix starting with module code, by default its lowercase.
  10. $columns - list of columns with domains (\Object\Data\Domains) and types (\Object\Data\Types) and default values.
  11. $constraints - list of table constrains like primary keys and unique constraints.
  12. $indexes - list of additional indexes.
  13. $history - flag indicating that table uses history tables.
  14. $audit - flag indicating that we audit this table.
  15. $optimistic_lock - flag indicated that writes are protected by optimistic lock.
  16. $options_map - array of columns to be displayed in select boxes on frontend.
  17. $options_active - indicates which columns needs to be skipped for frontend selects to only select active records.
  18. $engine - table engine for MySQL.
  19. $cache, cache_tags and cache_memory - indicated whether we need to cached and what are the tags and whether we need to cached in memory.
  20. $data_asset - classification, protection and scope of the table model.
  21. And more...

Explanation of columns settings:

  1. key - column database name.
  2. name - column model name.
  3. domain - column domain form \Object\Data\Domains, if set it presets type.
  4. type - column type, if domain is not set this is mandatory from \Object\Data\Types.
  5. null - boolean if null.
  6. options_model - optionally column can be a key in other \Object\Data model.
