ORM Tables

Table classes reside in Model folder of a module or application. Changes to these classes generate changes to database objects in form of migration or direct schema changes.

For example if you use migrations and want to add a new column to the existing table you would need to modify columns property and then run:

        
            make migration_code_test verbose=1 # view changes
            make migration_code_commit # generate migration files
            make migration_db_commit # commit db changes
