Main Application

There are few ways to develop application using Numbers Framework:

  1. Application Folders - you can put all necessary files into application folder and can start building your classes and models. More detailed is here.
  2. Private Modules - you can create a self contained module and put into /libraries/private/ folder. Each module contains everything it needs to run. More detailed is here.
  3. Public Modules - you can create a self contained module and post it using composer and then include it in application.ini file. More detailed is here.

Either way you choose to develop it would have pros and cons. For example:

  1. For application folders - you cannot port it to other projects and code cannot be reused. But its the fastest way to develop.
  2. Private Modules - this is preferred way to develop since the code is portable to other projects. You can just copy module folder and include it in application.ini file and that would be it. Use this method if you need to keep code private.
  3. Public Modules - this is preferred for commonly used open source modules. These can be installed by composer install and are public.

There's skeleton public repository available to preset application:

        
            composer create-project --no-install numbers/skeleton [project directory]

            # for example

            composer create-project --no-install numbers/skeleton my-test-project

            # then build project

            make build
